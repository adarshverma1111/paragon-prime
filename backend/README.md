# Paragon Prime Infotech — Contact Form Backend

Node.js + Express + MongoDB + SendGrid backend for both contact forms.

| Method | Endpoint              | Form                  |
|--------|-----------------------|-----------------------|
| POST   | `/api/contact/popup`  | ConsultationPopup     |
| POST   | `/api/contact/page`   | Contact Page          |

```json
  "fullName": "Adarsh Verma",

  "phone":    "+91 9876543210",
  "query":    "I need a mobile app"
```

```json
{ "success": true, "message": "Query received. We will contact you shortly." }
```

## Step-by-Step Setup

### 1. Install dependencies
```bash
npm install
```

### 2. Create your .env file
```bash
cp .env.example .env
```
Open `.env` and fill in all values (see comments inside).

### 3. MongoDB Atlas setup
1. Go to https://cloud.mongodb.com → create a free cluster.
2. Create a database user with read/write access.
3. Whitelist your server IP (or `0.0.0.0/0` for development).
4. Copy the connection string into `MONGO_URI`.

### 4. SendGrid setup
1. Go to https://app.sendgrid.com → **Settings → API Keys → Create API Key**.
2. Choose **Restricted Access**, enable **Mail Send**.
3. Paste the key into `SENDGRID_API_KEY`.

### 5. Verify your sender identity in SendGrid (REQUIRED)
SendGrid will reject mail from unverified senders.

> **Recommended: Domain Authentication** (not just single sender)

1. In SendGrid: **Settings → Sender Authentication → Authenticate Your Domain**.
2. Enter `paragonprimeinfotech.com`.
3. SendGrid gives you 3 CNAME records — add them to your domain's DNS.
4. Click **Verify** in SendGrid after DNS propagates (up to 48 h).

## ✅ Anti-Spam Checklist

Follow **all** steps below to stay out of Gmail / Outlook spam.

### A — DNS Records (most important)

Add these records in your domain registrar (GoDaddy / Cloudflare / etc.):

#### 1. SPF record (authorises SendGrid to send on your behalf)
| Type | Name | Value |
|------|------|-------|
| TXT  | `@`  | `v=spf1 include:sendgrid.net ~all` |

> If you already have an SPF record, merge it:
> `v=spf1 include:sendgrid.net include:your-existing ~all`

#### 2. DKIM records (generated by SendGrid during domain auth)
SendGrid automatically provides two CNAME records like:
```
em1234.paragonprimeinfotech.com  CNAME  em1234.dkim.sendgrid.net
s1._domainkey.paragonprimeinfotech.com  CNAME  s1.domainkey.u1234567.wl.sendgrid.net
```
Add them exactly as given by SendGrid.

#### 3. DMARC record (tells receivers what to do with failing mail)
| Type | Name    | Value |
|------|---------|-------|
| TXT  | `_dmarc`| `v=DMARC1; p=none; rua=mailto:connect@paragonprimeinfotech.com` |

Start with `p=none` (monitor mode). After 2 weeks with no issues,
upgrade to `p=quarantine`, then `p=reject`.

#### 4. Verify with MXToolbox
Go to https://mxtoolbox.com/SuperTool.aspx and check:
- SPF Lookup
- DKIM Lookup
- DMARC Lookup

All three should show **green / pass**.

### B — SendGrid Account Settings

| Setting | Recommended value |
|---------|-------------------|
| Domain Authentication | ✅ Complete (CNAME method) |
| Two-Factor Auth | ✅ Enable on your account |
| Dedicated IP | Optional — only needed above ~100 emails/day |
| Suppression Groups | Add an "unsubscribe" group for transactional mail |
| From address | Must match authenticated domain (`@paragonprimeinfotech.com`) |

### C — Email Content Best Practices (already applied in templates)

| Practice | Status |
|----------|--------|
| Plain-text version sent alongside HTML | ✅ |
| Preheader text set | ✅ |
| No spam trigger words ("free", "act now", "guaranteed") | ✅ |
| Physical mailing address in footer | ✅ |
| Unsubscribe instruction in footer | ✅ |
| `List-Unsubscribe` header | ✅ |
| Unique `X-Entity-Ref-ID` per email | ✅ |
| Click-tracking disabled (tracked links look like redirects) | ✅ |
| `role="presentation"` on layout tables | ✅ |
| Balanced image-to-text ratio (text-heavy) | ✅ |
| Short, clear subject line (no ALL CAPS / !!!) | ✅ |

### D — Frontend .env update

Update your Vite frontend `.env` to point to the right endpoints:

```env
# Popup form
VITE_CONTACT_ENDPOINT=https://your-backend.com/api/contact/popup

# Contact page form
VITE_CONTACT_PAGE_ENDPOINT=https://your-backend.com/api/contact/page
```

In `Contact.jsx`, change the endpoint:
```js
const endpoint =
  import.meta.env.VITE_CONTACT_PAGE_ENDPOINT ||
  "http://localhost:5000/api/contact/page";
```

## Running the Server

